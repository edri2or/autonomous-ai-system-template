# GitHub App Manifest Flow with Cloud Shell Web Preview

**Evidence Status:** Currently Proven  
**Date:** 2026-04-29  
**Source:** GitHub API documentation + Cloud Shell documentation

## Problem

Traditional GitHub App creation requires:
1. Manual navigation to GitHub Settings
2. Downloading a `.pem` file to the browser
3. Manual upload to GCP Secret Manager
4. High risk of credential exposure during transfer

**Goal:** Automate credential creation and storage without any file downloads or manual credential pasting.

## Solution: Manifest Flow + Cloud Shell Web Preview

### How It Works (3 Steps)

#### Step 1: Script Prepares Manifest + HTTP Server
- Generate GitHub App manifest as JSON (name, permissions, events, redirect_url)
- Calculate `redirect_url` using Cloud Shell's `$WEB_HOST` environment variable
- Start Python HTTP server on port 8080 listening for redirect callback
- Display clickable URL to user (manifest encoded in query string)

#### Step 2: User Clicks (2 Actions)
1. Click the URL → GitHub shows "Create GitHub App" button → click "Create"
2. GitHub shows "Install App" → click "Install"

#### Step 3: Script Completes Everything Automatically
- Python server receives `GET /callback?code=xxx&state=yyy` from GitHub redirect
- Extract code and pass to GitHub API: `POST /app-manifests/{code}/conversions`
- Response contains `id`, `pem`, `webhook_secret`, `client_id`, `client_secret`
- Store all credentials in GCP Secret Manager using `gcloud secrets`
- Delete Python server
- Cleanup complete — no local files, no credential exposure

### Technical Details

**Cloud Shell Web Preview URL Format:**
```
https://{PORT}-{$WEB_HOST}
```
- `$WEB_HOST` is an environment variable automatically set in Cloud Shell
- Port range: 2000–65000
- Example: `https://8080-cs-12345678-default.cs-europe-west1-iuzs.cloudshell.dev/`

**Manifest Structure:**
```json
{
  "name": "{ORG}-agent",
  "url": "https://github.com/{ORG}",
  "hook_attributes": {
    "url": "https://placeholder.example.com",
    "active": false
  },
  "redirect_url": "https://8080-$WEB_HOST/callback",
  "public": false,
  "default_permissions": {
    "contents": "write",
    "issues": "write",
    "pull_requests": "write",
    "actions": "write",
    "secrets": "write",
    "workflows": "write",
    "metadata": "read"
  },
  "default_events": ["push", "pull_request"]
}
```

**Code Exchange Endpoint:**
```
POST https://api.github.com/app-manifests/{CODE}/conversions
```

**Response Fields:**
- `id` (GitHub App ID) — integer
- `pem` (private key) — string (full PEM content)
- `webhook_secret` — string
- `client_id` — string
- `client_secret` — string

**Time Limit:** Code is valid for 1 hour after creation.

**State Parameter:** Security token generated by script, verified in redirect URL to prevent CSRF.

### Proven Implementation Pattern

This pattern is proven in production by:
- GitHub's official manifest flow documentation
- Cloud Shell's native web preview capability
- Standard OAuth/manifest code exchange flow (used by Probot and other GitHub App frameworks)

### Security Properties

✅ **No credential exposure:**
- PEM never touches the user's filesystem
- PEM never printed to terminal
- PEM never copied into chat
- Stored directly in GCP Secret Manager via piped stream

✅ **Minimal user action:**
- Only 2 clicks required (both on GitHub UI)
- Everything else is automated

✅ **Cloud-native:**
- Works in Cloud Shell without local machine
- No installation of tools needed beyond `python3`
